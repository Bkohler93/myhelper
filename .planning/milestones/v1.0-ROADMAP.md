# Roadmap: myhelper

**Milestone:** v1.0 — Initial Release
**Goal:** Deliver a fully functional Go CLI that runs five focused coding-assistance commands against a local Ollama model, with project context scoping via `context.md` and streaming output.

## Phase 1: Full Implementation

**Goal:** All five commands work end-to-end — reading config and `context.md`, constructing focused prompts, streaming Ollama responses to stdout — installable as a single binary.
**Requirements:** CMD-01, CMD-02, CMD-03, CMD-04, CMD-05, CMD-06, CTX-01, CTX-02, CTX-03, MDL-01, MDL-02, MDL-03, MDL-04, PRM-01, PRM-02, PRM-03

### Plans

#### Plan 1.1: Project Scaffold and Ollama Client
Set up the Go module structure, cobra root command, and a reusable Ollama client package. The client handles streaming HTTP requests to the `/api/chat` endpoint and writes tokens to stdout as they arrive. Config loading (file or environment variable, with hardcoded defaults for endpoint and model) lives here as a shared concern consumed by every command.
**Delivers:** MDL-01, MDL-02, MDL-03, MDL-04

#### Plan 1.2: Context System
Implement the `context.md` reader that locates the file in the current working directory, loads its contents, and returns an empty string gracefully when the file is absent. Write the `init` command, which writes a commented template skeleton to `context.md` without overwriting an existing one. The context string is the sole shared input injected into every prompt.
**Delivers:** CTX-01, CTX-02, CTX-03, CMD-01

#### Plan 1.3: Command Framework and Argument Handling
Wire up the four remaining subcommands (`plan`, `lookup`, `starter`, `pattern`) under the cobra root. Each command reads its positional argument; if the argument is omitted, it prompts the user interactively before proceeding. Commands delegate to the Ollama client and context reader established in Plans 1.1 and 1.2.
**Delivers:** CMD-02, CMD-03, CMD-04, CMD-05, CMD-06

#### Plan 1.4: Prompt Design
Write the distinct system prompts for each command: `plan` (decompose into ordered subtasks), `lookup` (recommend the right API or library), `starter` (print minimal working Go code), `pattern` (describe idiomatic Go structure). Each prompt is kept minimal so that system prompt + context.md + user input fits within 8k tokens. All prompts are single-shot — no conversation history is included.
**Delivers:** PRM-01, PRM-02, PRM-03
