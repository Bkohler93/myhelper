# Requirements: myhelper v1.1

**Defined:** 2026-04-07
**Core Value:** Get a useful, context-aware answer from the local model in one command, without context-bloat or round-trips to an external API.

## v1.1 Requirements

### Conversation Loop

- [x] **CONV-01**: After the first model response, user is prompted for a follow-up input without restarting the command
- [x] **CONV-02**: User can type "quit" to end the conversation session cleanly
- [x] **CONV-03**: User can press Ctrl+C to end the conversation session cleanly
- [x] **CONV-04**: Conversation loop applies to plan, lookup, starter, and pattern commands

### History Management

- [x] **HIST-01**: Each user turn and model response is appended to an in-memory conversation history
- [x] **HIST-02**: Token count of the conversation history is tracked after each turn using go-tiktoken
- [x] **HIST-03**: When history token count exceeds the threshold, summarization is triggered before the next model call

### Summarization

- [x] **SUMM-01**: When the threshold is reached, the model is called with a prompt instructing it to summarize the conversation history, prioritizing content relevant to the original goal
- [x] **SUMM-02**: The full history is replaced with the compact summary after summarization
- [x] **SUMM-03**: On subsequent threshold hits, the prior summary and new turns are condensed together into a new summary (re-condensation)

### Configuration

- [x] **CONF-01**: Token threshold is configurable via an environment variable or CLI flag (default: 4,100)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Conversation history persistence across sessions | Single-session only; stateless between invocations matches existing design |
| `init` command conversation mode | init writes a file and exits — no model response to follow up on |
| Multi-model conversation routing | One model per session; model config unchanged from v1.0 |
