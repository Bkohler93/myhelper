# Requirements: myhelper

**Defined:** 2026-04-07
**Core Value:** Get a useful, context-aware answer from the local model in one command, without context-bloat or round-trips to an external API.

## v1 Requirements

### CLI Commands

- [x] **CMD-01**: `myhelper init` writes a blank `context.md` template to the current directory
- [x] **CMD-02**: `myhelper plan <description>` breaks a feature/task into ordered subtasks
- [x] **CMD-03**: `myhelper lookup <question>` recommends the right API or library for a step
- [x] **CMD-04**: `myhelper starter <task>` prints minimal working Go code for a given task
- [x] **CMD-05**: `myhelper pattern <topic>` describes the idiomatic Go way to write or structure something
- [x] **CMD-06**: All commands accept input as a CLI argument; prompt interactively if argument is omitted

### Context System

- [x] **CTX-01**: `context.md` in the current directory is read and prepended to every model prompt
- [x] **CTX-02**: `init` writes a commented template skeleton — user fills it in manually, nothing is auto-detected
- [x] **CTX-03**: If `context.md` is absent, commands still work (model uses no project context)

### Model Integration

- [x] **MDL-01**: All commands stream model output to stdout as tokens arrive (no buffering)
- [x] **MDL-02**: Ollama endpoint is configurable (default: `192.168.0.9:11434`)
- [x] **MDL-03**: Model name is configurable (default: `qwen2.5-coder:7b`)
- [x] **MDL-04**: Config is read from a local config file or environment variable; hardcoded defaults used as fallback

### Prompt Design

- [x] **PRM-01**: Each command uses a distinct, focused system prompt appropriate to its task
- [x] **PRM-02**: Prompts are kept minimal — system prompt + context.md + user input must fit within 8k tokens
- [x] **PRM-03**: Prompts are single-shot (no conversation history sent to model)

## v2 Requirements

### Quality of Life

- **UX-01**: `--model` and `--endpoint` flags to override config per-invocation
- **UX-02**: `myhelper config` command to view/set endpoint and model
- **UX-03**: Shell completion (bash/zsh) for command names

## Out of Scope

| Feature | Reason |
|---------|--------|
| Auto-detect project metadata on init | Pure template keeps init fast and predictable; user knows their stack |
| Global fallback context.md | Per-directory only avoids cross-project bleed |
| Multi-turn conversation history | Blows the 8k context budget; single-shot is the constraint |
| Non-Go project support | Tool is optimized for Go; other languages are incidental |
| External API calls or web search | Local inference only |
| TUI / interactive chat interface | Command-oriented, not chat-oriented |
