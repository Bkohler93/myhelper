### Phase 28 — Distribution

- **install.sh archive extraction path** — `cp "${TMP}/${BINARY}"` assumes binary at archive root. goreleaser v2 `wrap_in_directory` is not explicitly set to `false` in `.goreleaser.yaml`. Default behavior should produce binary at root, but this has not been verified against a real snapshot build. Recommended: run `goreleaser release --snapshot` before first tag push and confirm `dist/` archive structure. Fix if needed: add `wrap_in_directory: false` to `.goreleaser.yaml` archives block, or change cp path to `"${TMP}/${ARCHIVE%.tar.gz}/${BINARY}"`.
