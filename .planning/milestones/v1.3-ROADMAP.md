## Overview

Transformed context handling from static injection into a structured, navigable representation of the codebase. The model now retrieves incrementally (symbol → file → dependency expansion) rather than guessing context upfront. Replaced the flat `index.json` with four layered artifact files, added a full four-stage LLM retrieval pipeline, wired per-command retrieval strategies across all query commands, and added the `inspect` dry-run diagnostic command.

## Milestone Summary

**Key Decisions:**
- Four-stage retrieval pipeline chosen over vector/embedding approach — structured retrieval sufficient for ~200-500 symbol corpus
- Fail-open gate: context omission is worse than extra tokens for a 7B model
- Small corpus (≤40 files) path bypasses pre-filter hard gate — all symbols pass as additive hints capped by budget
- `microPassFile` moved to `internal/retrieval/` — enforces retrieval-logic-in-retrieval-package rule

**Issues Resolved:**
- `buildInjectedMessages` replaced and deleted — no dead retrieval code remaining in `cmd/`
- `ApplyFlagOverrides` wired into all 4 query commands + inspect — closes v1.2 known gap
- `--token-limit` now correctly caps retrieval budget calculations (was silently no-op in v1.2)

**Issues Deferred:**
- Phase 11 VERIFICATION.md never produced — RET-01–06 formally "partial" (process gap, not implementation gap)
- `Symbol.CallEdges`/`TypeRefs` stored but not consumed by retrieval pipeline (SYM-05, SYM-06)
- `PackageEntry.Responsibility` written to `packages.json` but unused in `llmReRank` (IDX-03, RET-03)
- `Symbol.Start/End` stored in artifact but `microPassFile` re-parses AST at runtime (SYM-03)
- Dual context injection (`context.md` + `proj.Summary`) — same source, redundant tokens (CTX-01, CTX-02)
- `inspect` ignores `--no-context` flag (WR-04 from Phase 13 code review)
- `BuildContext`/`BuildInspectContext` silently discard `llmReRank` error return (WR-02)

**Technical Debt Incurred:**
- `pkgs []scanner.PackageEntry` parameter in `llmReRank` — declared but never referenced in body
- `ErrStaleFlatIndex` has no production caller — test-only path
- `FileArtifactEntry.ExportedNames` written to files.json but not read by retrieval pipeline
- `cmd/inspect.go` passes `DefaultStrategy` to `BuildInspectContext` — strategy is a dead parameter
