**Step B — BUG-02: update both call sites to use named error variables.**

Call site 1 — BuildContext (line ~124):

Change:
```go
selected, _ := llmReRank(query, candidates, pkgs.Packages, cfg, chatFn)
```

To:
```go
selected, err := llmReRank(query, candidates, cfg, chatFn)
if err != nil {
    selected = candidates // fallback: use all candidates on LLM error
}
```

Note: `err` may shadow an outer `err` variable. If a compile error occurs, use `reRankErr` instead:
```go
selected, reRankErr := llmReRank(query, candidates, cfg, chatFn)
if reRankErr != nil {
    selected = candidates
}
```

Call site 2 — BuildInspectContext (line ~843):

Change:
```go
selected, _ := llmReRank(query, candidates, pkgs.Packages, cfg, chatFn)
```

To:
```go
selected, reRankErr := llmReRank(query, candidates, cfg, chatFn)
if reRankErr != nil {
    selected = candidates // fallback
}
```

After both changes, `pkgs.Packages` is no longer passed to llmReRank. The `pkgs` variable is still used elsewhere in BuildContext (e.g. `expandDeps` call) so do NOT remove the `pkgs` variable itself.
