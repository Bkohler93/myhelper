Create `install.sh` at the project root with the following exact content. This script implements the DIST-01 requirement for single-command install.

```bash
#!/usr/bin/env bash
# install.sh — myhelper installer
# Usage: curl -sfL https://raw.githubusercontent.com/bkohler93/myhelper/main/install.sh | bash
#
# Environment overrides:
#   INSTALL_DIR   Override install directory (default: ~/.local/bin)
#
# Examples:
#   curl -sfL https://raw.githubusercontent.com/bkohler93/myhelper/main/install.sh | bash
#   curl -sfL https://raw.githubusercontent.com/bkohler93/myhelper/main/install.sh | INSTALL_DIR=/usr/local/bin bash
set -euo pipefail

REPO="bkohler93/myhelper"
BINARY="myhelper"
INSTALL_DIR="${INSTALL_DIR:-${HOME}/.local/bin}"

# ── OS / arch detection ────────────────────────────────────────────────────────
uname_os() {
  local os
  os=$(uname -s | tr '[:upper:]' '[:lower:]')
  case "$os" in
    mingw*) os="windows" ;;
    cygwin*) os="linux" ;;
  esac
  echo "$os"
}

uname_arch() {
  local arch
  arch=$(uname -m)
  case "$arch" in
    x86_64)  arch="amd64" ;;
    aarch64) arch="arm64" ;;
    armv7*)  arch="arm" ;;
  esac
  echo "$arch"
}

OS=$(uname_os)
ARCH=$(uname_arch)

# ── Fetch latest release tag ───────────────────────────────────────────────────
TAG=$(curl -sf "https://api.github.com/repos/${REPO}/releases/latest" \
  | grep '"tag_name"' | head -1 | sed 's/.*"tag_name": "\(.*\)".*/\1/')

if [[ -z "$TAG" ]]; then
  echo "Error: could not determine latest release tag from GitHub API." >&2
  echo "  Check https://github.com/${REPO}/releases for the latest version." >&2
  exit 1
fi

VERSION="${TAG#v}"   # strip leading 'v' — archive names use "1.0.0" not "v1.0.0"

# ── Construct download URLs ────────────────────────────────────────────────────
BASE_URL="https://github.com/${REPO}/releases/download/${TAG}"
ARCHIVE="${BINARY}_${VERSION}_${OS}_${ARCH}.tar.gz"
CHECKSUM="${BINARY}_${VERSION}_checksums.txt"

echo "Installing ${BINARY} ${TAG} (${OS}/${ARCH})..."

# ── Download to temp dir ───────────────────────────────────────────────────────
TMP=$(mktemp -d)
trap 'rm -rf "$TMP"' EXIT

curl -sfL "${BASE_URL}/${ARCHIVE}" -o "${TMP}/${ARCHIVE}"
curl -sfL "${BASE_URL}/${CHECKSUM}" -o "${TMP}/${CHECKSUM}"

# ── Verify checksum ────────────────────────────────────────────────────────────
cd "$TMP"
if command -v sha256sum >/dev/null 2>&1; then
  sha256sum --ignore-missing -c "${CHECKSUM}"
elif command -v shasum >/dev/null 2>&1; then
  # macOS ships shasum, not sha256sum
  grep "${ARCHIVE}" "${CHECKSUM}" | shasum -a 256 -c -
else
  echo "Warning: sha256sum and shasum not found — skipping checksum verification." >&2
fi
cd - >/dev/null

# ── Extract and install ────────────────────────────────────────────────────────
tar -xzf "${TMP}/${ARCHIVE}" -C "${TMP}"
mkdir -p "${INSTALL_DIR}"
cp "${TMP}/${BINARY}" "${INSTALL_DIR}/${BINARY}"
chmod +x "${INSTALL_DIR}/${BINARY}"

echo "Installed ${BINARY} ${TAG} to ${INSTALL_DIR}/${BINARY}"

# ── PATH hint ─────────────────────────────────────────────────────────────────
if [[ ":$PATH:" != *":${INSTALL_DIR}:"* ]]; then
  echo ""
  echo "NOTE: ${INSTALL_DIR} is not in your PATH."
  echo "Add this to your ~/.bashrc or ~/.zshrc:"
  echo "  export PATH=\"\$HOME/.local/bin:\$PATH\""
fi
```

After writing the file, make it executable:


Key design decisions in this script:
- `set -euo pipefail` — script aborts immediately on any error, unset variable use, or pipe failure
- `INSTALL_DIR="${INSTALL_DIR:-${HOME}/.local/bin}"` — defaults to ~/.local/bin (no sudo), but user can override with `INSTALL_DIR=/usr/local/bin bash`
- `uname -s | tr '[:upper:]' '[:lower:]'` — converts "Linux" → "linux", "Darwin" → "darwin" to match goreleaser's `{{ .Os }}` template output (always lowercase)
- `uname -m` + case statement — converts "x86_64" → "amd64", "aarch64" → "arm64" to match goreleaser's `{{ .Arch }}` output
- `VERSION="${TAG#v}"` — goreleaser archive names use "1.0.0" (no v prefix) while the GitHub API returns "v1.0.0" as the tag_name
- `curl -sfL` — `-s` silent, `-f` fail on HTTP errors (without -f, curl returns 0 on 404 and the script would install garbage), `-L` follow redirects
- `sha256sum --ignore-missing` — verifies only the archive file without failing on the other entries in the multi-file checksums.txt
- `shasum -a 256` fallback — macOS does not ship sha256sum; shasum is the equivalent
- `trap 'rm -rf "$TMP"' EXIT` — temp directory cleanup guaranteed even if script exits early due to set -e
- ARCHIVE naming: `${BINARY}_${VERSION}_${OS}_${ARCH}.tar.gz` — must exactly match goreleaser's name_template `{{ .ProjectName }}_{{- .Version }}_{{- .Os }}_{{- .Arch }}.tar.gz`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → user machine | curl downloads binary from github.com and checksums file from same release |
| GitHub API → install.sh | Unauthenticated API call to determine latest release tag |
| INSTALL_DIR env var → mkdir/cp | User-controlled path used for install location |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-08 | Tampering | Downloaded binary | mitigate | sha256sum verification against checksums.txt (both from GitHub Releases via TLS); detects corruption and download errors |
| T-28-09 | Tampering | curl pipe execution | accept | Script is public on GitHub; user can inspect before running: `curl -sfL <url> | less`; standard industry practice for Go tool install scripts |
| T-28-10 | Tampering | tar slip (path traversal) | mitigate | Binary extracted to `$TMP` (mktemp -d), then explicitly copied to `$INSTALL_DIR/$BINARY`; no `--strip-components` or wildcard paths |
| T-28-11 | Denial of Service | GitHub API rate limiting | accept | 60 req/hour per IP for unauthenticated; individual users rarely hit this; error message directs user to releases page |
| T-28-12 | Elevation of Privilege | INSTALL_DIR injection | mitigate | INSTALL_DIR only used in `mkdir -p` and `cp` — no eval, no command injection possible; worst case: user installs to wrong directory |
| T-28-13 | Information Disclosure | install.sh shows OS/arch | accept | OS and arch are displayed in progress output — not sensitive; aids debugging |
</threat_model>
