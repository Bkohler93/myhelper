Create `.goreleaser.yaml` at the project root with the following exact content. Every field is non-negotiable per research:

```yaml
version: 2

project_name: myhelper

before:
  hooks:
    - go mod tidy

builds:
  - env:
      - CGO_ENABLED=0
    goos:
      - linux
      - darwin
    goarch:
      - amd64
      - arm64
    flags:
      - -trimpath
    ldflags:
      - -s -w -X github.com/bkohler93/myhelper/cmd.Version={{.Version}} -X github.com/bkohler93/myhelper/cmd.Commit={{.Commit}} -X github.com/bkohler93/myhelper/cmd.Date={{.Date}}
    mod_timestamp: "{{ .CommitTimestamp }}"

archives:
  - name_template: >-
      {{ .ProjectName }}_
      {{- .Version }}_
      {{- .Os }}_
      {{- .Arch }}
    formats:
      - tar.gz

checksum:
  name_template: "{{ .ProjectName }}_{{ .Version }}_checksums.txt"
  algorithm: sha256

changelog:
  sort: asc
  filters:
    exclude:
      - "^docs:"
      - "^test:"
      - "^chore:"

release:
  github:
    owner: bkohler93
    name: myhelper
```

Key decisions embedded in this config:
- `version: 2` — required for goreleaser v2.x; omitting causes an error
- `CGO_ENABLED=0` — required for cross-compilation from ubuntu-latest to darwin targets (macOS C toolchain not available on Linux runner)
- `-trimpath` — removes local build paths from binary for reproducibility
- `-s -w` — strips debug info and DWARF symbols (~30% smaller binary)
- ldflags use `github.com/bkohler93/myhelper/cmd.Version` (not `main.version`) — the variable is declared in the `cmd` package (Task 2), not in main
- `formats: [tar.gz]` — plural form is goreleaser v2.6+ forward-compatible
- `mod_timestamp: "{{ .CommitTimestamp }}"` — enables reproducible builds
- archive `name_template` uses `{{ .Os }}` which goreleaser lowercases — this matches install.sh's `uname -s | tr '[:upper:]' '[:lower:]'` output
