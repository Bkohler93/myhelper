### Phase 27: Inspect Rewrite

**Goal**: `myhelper inspect <query>` is a useful web search diagnostic that shows exactly what the search pipeline would do for a given query
**Depends on**: Phase 26
**Requirements**: INSP-01, INSP-02, INSP-03, INSP-04, INSP-05, INSP-06, INSP-07
**Plans**: 1 plan

Plans:
- [x] 27-01-PLAN.md — full inspect rewrite: gate output, fetch output, re-rank groups, injected block preview, token cost

**Success Criteria:**
1. Running `myhelper inspect <query>` prints the gate LLM's YES/NO decision and raw answer text
2. When gate says NO, inspect prints "search not needed" and exits without fetching results
3. When gate says YES, inspect prints every fetched SearXNG result with title, URL, and snippet
4. Inspect prints re-rank output as `Survivors (N):` and `Dropped (N):` labeled groups
5. Inspect prints the full `[WEB RESULTS]` block that would be injected, followed by its token count
6. `--search` forces inspect to bypass the gate and run the complete pipeline regardless of gate decision
7. `--no-search` causes inspect to print "search suppressed" and exit immediately

## Milestone Summary

**Key Decisions:**
- Delete dead retrieval pipeline: v4.0 direction is chat+web-search-first; the `.myhelper/` retrieval packages (context/planner/retrieval/scanner) had no live callers after v3.0 and added ~5,500 lines of dead weight
- Rewrite inspect as web search diagnostic: the old inspect showed retrieval pipeline decisions which are now moot; the new inspect makes real LLM+SearXNG calls to show the actual gate/fetch/re-rank path
- `--search` in inspect always runs re-rank: diagnostic parity — inspect `--search` should show the full pipeline the same as gate=YES, even though chat mode with `--search` skips re-rank

**Issues Resolved:**
- Dead package drift: internal/retrieval was imported only by the old inspect stub; once replaced, deletion of all four packages was clean with zero live callers remaining
- `--no-context` flag dangling reference: flag was only meaningful alongside the retrieval pipeline; removed atomically with the packages

**Issues Deferred:**
- CLAUDE.md architecture docs still describe the deleted packages — documentation drift noted as tech debt
- Missing VALIDATION.md for both phases (Nyquist compliance gap — process debt only, both phases verified by other means)

**Technical Debt Incurred:**
- `llmReRank` always returns nil error by design — named `reRankErr` branches are technically dead code in `cmd/search.go`
- REQUIREMENTS.md traceability checkboxes were not updated during development (all archived as-is with Pending status, actual status is satisfied)
