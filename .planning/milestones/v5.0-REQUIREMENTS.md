## v5.0 Requirements

### Distribution

- [x] **DIST-01**: User can install myhelper on WSL/Linux by running a one-line curl command that auto-detects OS/arch, downloads the correct binary from GitHub Releases, and places it in PATH
  - **Outcome:** install.sh implemented with uname OS/arch detection, sha256 verification, ~/.local/bin install, PATH hint
  - **Note:** End-to-end test deferred until first real GitHub release

- [x] **DIST-02**: Tagged git pushes automatically trigger GitHub Actions to build cross-platform binaries (darwin/amd64, darwin/arm64, linux/amd64, linux/arm64) and publish them to GitHub Releases via goreleaser
  - **Outcome:** .goreleaser.yaml + .github/workflows/release.yml wired; goreleaser-action@v7 on v* tag push

### Setup Wizard

- [x] **SETUP-01**: User can run `myhelper setup` to check whether Ollama is installed and reachable on port 11434
  - **Outcome:** checkOllama() in wizard.go probes localhost:11434; cmd/setup.go registers cobra subcommand

- [x] **SETUP-02**: User sees platform-specific Ollama install instructions when Ollama is not detected (macOS: `brew install ollama`, Linux/WSL: curl install script)
  - **Outcome:** installInstructions() switches on runtime.GOOS; human-validated 2026-05-10

- [x] **SETUP-03**: User sees a recommended model size based on detected GPU VRAM (nvidia-smi on Linux/WSL, system_profiler on macOS) or RAM when no discrete GPU is found
  - **Outcome:** detectMemoryMiB() with nvidia-smi/system_profiler/proc/meminfo chain; 4-tier recommendModel() table

- [x] **SETUP-04**: User can confirm to have the wizard run `ollama pull <recommended-model>` directly without leaving the terminal
  - **Outcome:** pullModel() streams NDJSON progress via /api/pull; human-validated 2026-05-10

- [x] **SETUP-05**: User is prompted to enter their Tavily API key during setup; key is written to `~/.config/myhelper/config.json`
  - **Outcome:** mergeHomeConfig({"tavily_key": ..., "search_provider": "tavily"}); 0600 perms; human-validated 2026-05-10

- [x] **SETUP-06**: User can optionally enter a SearXNG endpoint during setup; endpoint is written to config
  - **Outcome:** http(s) prefix validation; mergeHomeConfig({"search_endpoint": ...}); human-validated 2026-05-10

### Search Providers

- [x] **SRCH-01**: User can use Tavily as the search provider by configuring a Tavily API key in config or via env var; Tavily is the default provider when a key is present
  - **Outcome:** tavilySearch POST + Bearer auth; LoadConfig auto-selects "tavily" when TavilyKey≠""

- [x] **SRCH-02**: User can switch search provider between Tavily and SearXNG via `search_provider` field in config.json
  - **Outcome:** Config.Provider with json:"search_provider" tag; Search() dispatches by Provider field

- [x] **SRCH-03**: User can provide their Tavily API key via `MYHELPER_TAVILY_KEY` environment variable, which takes precedence over config
  - **Outcome:** LoadConfig applies MYHELPER_TAVILY_KEY after config file layers; MYHELPER_SEARCH_PROVIDER also supported
