## v1.3 Requirements

### Index Structure

- [x] **IDX-01**: `init` and `sync` produce four hierarchical artifact files: `project.json`, `packages.json`, `files.json`, `symbols.json` — **COMPLETE** (Phase 10)
- [x] **IDX-02**: `project.json` contains project-level metadata (module path, Go version, total file/symbol counts) and an LLM-generated one-paragraph project summary — **COMPLETE** (Phase 10)
- [x] **IDX-03**: `packages.json` contains each package's import path, list of files, and a brief responsibility description — **COMPLETE** (Phase 10; responsibility stored, unused in re-rank — tech debt)
- [x] **IDX-04**: `files.json` contains each file's path, package, list of exported symbol names, and resolved import paths — **COMPLETE** (Phase 10)
- [x] **IDX-05**: `symbols.json` contains each exported symbol's name, kind, full signature, receiver, source file, line range, call edges, and references per symbol — **COMPLETE** (Phase 10)
- [x] **IDX-06**: Index artifacts include a schema version field; stale flat `index.json` consumers return a typed error (`ErrStaleFlatIndex`) — **COMPLETE** (Phase 10)

- [x] **SYM-04**: `go/ast` extraction captures the import paths declared in each file — **COMPLETE** (Phase 9)

### Retrieval Pipeline

- [x] **RET-01**: Deterministic keyword/symbol pre-filter narrows the full symbol corpus to a candidate set before LLM evaluation — ⚡ **COMPLETE** (Phase 11; implementation confirmed; VERIFICATION.md missing — process gap only)
- [x] **RET-02**: Pre-filter operates as an additive hint when corpus is small (≤ ~40 files), capped by token threshold — ⚡ **COMPLETE** (Phase 11; `smallCorpusThreshold=40`; same process gap)
- [x] **RET-03**: LLM re-ranking pass evaluates pre-filtered candidates using summaries and signatures — ⚡ **COMPLETE** (Phase 11; same process gap)
- [x] **RET-04**: Context relevance gate detects when repository context is not needed and skips retrieval entirely — ⚡ **COMPLETE** (Phase 11; fail-open design; same process gap)
- [x] **RET-05**: Dependency-aware expansion includes depth-1 import graph files, bounded at ≤ 60% of remaining token budget — ⚡ **COMPLETE** (Phase 11; `expansionBudgetFactor=0.60`; same process gap)
- [x] **RET-06**: All retrieval logic in `internal/retrieval/` behind `BuildContext` entry point; no retrieval logic in `cmd/` or `helpers.go` — ⚡ **COMPLETE** (Phase 11/12; confirmed by Phase 12 VERIFICATION.md)

- [x] **CTX-03**: `buildInjectedMessages` in `helpers.go` replaced by `retrieval.BuildContext`; old function deleted — **COMPLETE** (Phase 12)

## Out of Scope (Confirmed)

| Feature | Reason |
|---------|--------|
| Vector/embedding search | Structured retrieval is preferred; avoids unnecessary complexity |
| Iterative retrieval (default on) | P2 — adaptive builder makes it largely unnecessary for common cases |
| Nested sub-indexes | Revisit if hierarchical index exceeds token budget at larger project scale |
| Cross-project / global context | Per-directory only, avoids cross-project bleed |
| Non-Go file indexing | Tool remains Go-first |
| BM25 / TF-IDF scoring | Linear token-overlap sufficient for ~200-500 symbol corpus |
| Dynamic call resolution | No type-checker; interface satisfiers and dynamic dispatch out of scope for v1.3 |
