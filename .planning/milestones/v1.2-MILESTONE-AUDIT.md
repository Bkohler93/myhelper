### Notable Integration Gaps

#### Gap 1: `ApplyFlagOverrides` missing from all 4 query commands (non-critical) — STILL PRESENT

`ApplyFlagOverrides` is called in `cmd/init.go:33` and `cmd/sync.go:35` after `config.Load()`, but is absent from `cmd/plan.go:44`, `cmd/lookup.go:44`, `cmd/starter.go:44`, and `cmd/pattern.go:44`. The `--token-limit` flag is accepted by Cobra but silently ignored on all query commands. **Consequence:** `--token-limit N` on any query command has no effect on the 80% token budget in `buildInjectedMessages`, the `microPassFile` budget, or the conversation history limit. Affects CTX-01, CTX-03.

#### Gap 2: Sync guard behavior — EFFECTIVELY RESOLVED, edge case remains

The original gap (sync refusing when init is interrupted after Scan but before writeLastSync) does not apply because the meta.json guard is consistent with what writeLastSync writes. However: if a user manually deletes `.myhelper/index.json` while meta.json still exists, `deltaIndex` silently swallows the `json.Unmarshal` error and proceeds as if all files are new. Non-breaking, but a silent-failure mode. Affects SYNC-01, SYNC-02 (edge case only).

#### Gap 3: `generateContextMD` hard-errors on empty summaries (non-critical) — STILL PRESENT

`cmd/helpers.go:262` returns a hard error when `summaryCount == 0`. Reachable on projects with zero exported symbols. The query command fallback (`injectSummaries`) handles this gracefully — the inconsistency makes `init`/`sync` fragile on unusual projects. Affects INIT-01, SYNC-01.
