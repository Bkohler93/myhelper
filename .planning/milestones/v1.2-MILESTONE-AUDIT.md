### Notable Integration Gaps

#### Gap 1: `ApplyFlagOverrides` missing from all 4 query commands (non-critical)

`ApplyFlagOverrides` is called in `cmd/init.go:33` and `cmd/sync.go:35` after `config.Load()`, but is absent from `cmd/plan.go`, `cmd/lookup.go`, `cmd/starter.go`, and `cmd/pattern.go`. The `--token-limit` flag is a persistent flag on `rootCmd` and is populated by Cobra, but the override is never applied to `cfg` in any query command. **Consequence:** `--token-limit N` on any query command silently has no effect on the 80% token budget in `buildInjectedMessages` or the conversation history limit. Affects CTX-01.

#### Gap 2: Sync guard checks `meta.json` instead of `index.json` (non-critical)

`cmd/sync.go:38-41` refuses to run if `.myhelper/meta.json` does not exist. `meta.json` is written only by `writeLastSync` at the end of `runInit` — **after** `scanner.Scan` and `generateContextMD` succeed. If `init` is interrupted between `scanner.Scan` (which writes `index.json`) and `writeLastSync` (which writes `meta.json`), `sync` will refuse to run even though `index.json` is valid. The guard should check `index.json`. Affects SYNC-01, SYNC-02.

#### Gap 3: `generateContextMD` hard-errors on empty summaries (non-critical)

`cmd/helpers.go:262` returns an error when `summaryCount == 0`. If `scanner.GenerateSummaries` produces no `.md` files (e.g., a project with no exported symbols), `context.md` is never written and `init`/`sync` abort under `RunWithSpinner`. Affects INIT-01, SYNC-01.
