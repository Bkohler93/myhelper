### Integration Gaps

#### Gap 1: `ApplyFlagOverrides` missing from all 4 query commands (non-critical) — STILL PRESENT

`ApplyFlagOverrides` is called in `cmd/init.go:33` and `cmd/sync.go:35` after `config.Load()`, but is absent from `cmd/plan.go:44`, `cmd/lookup.go:44`, `cmd/starter.go:44`, and `cmd/pattern.go:44`. The `--token-limit` flag is accepted by Cobra but silently ignored on all query commands. `--token-limit N` on any query command has no effect on the 80% token budget in `buildInjectedMessages`, the `microPassFile` budget, or the conversation history limit. Affects CTX-01, CTX-03.

#### Gap 2: `generateContextMD` hard-errors on empty summaries (non-critical) — STILL PRESENT

`cmd/helpers.go:262-264` returns a hard error when `summaryCount == 0`. Reachable on projects with zero exported symbols. The query command fallback (`injectSummaries`) handles this gracefully — the inconsistency makes `init`/`sync` fragile on unusual projects. Affects INIT-01, SYNC-01.

#### Gap 3: `deltaIndex` silently swallows `json.Unmarshal` errors (non-critical) — STILL PRESENT

`cmd/sync.go:130-133` uses `_ = json.Unmarshal(data, &existing)` — if `index.json` is deleted while `meta.json` exists, proceeds as if all files are new. Silent data path. Affects SYNC-01, SYNC-02.

#### Gap 4: `deltaIndex` re-uses stale `Index.Meta` — `go.mod` changes not reflected after sync (non-critical) — NEW

`cmd/sync.go:197` writes `idx := scanner.Index{Meta: existing.Meta, Files: merged}`. `ReadMeta` is never called by sync. `changedFilesSince` (sync.go:98) only walks `.go` files, so a `go.mod`-only change triggers no delta at all. Even when `deltaIndex` runs on `.go` changes, the meta is always the stale version from the last `init`. `Index.Meta.ModuleName` and `ModuleDeps` in `index.json` will be stale after `go.mod` changes until the user runs `init` again. Affects INIT-03, SYNC-01.
