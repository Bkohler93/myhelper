<objective>
Add startup validation that hard-fails chat and inspect commands when model or endpoint is not configured. The validation runs after config.Load() + ApplyFlagOverrides() and before any Ollama calls. The error message always includes "myhelper setup" as the remediation hint. Cobra's double-print bug is suppressed via SilenceErrors=true.

Purpose: After Plan 01 removes hardcoded defaults, Load() returns empty strings for unconfigured fields. This plan detects those empty strings and surfaces a clear, actionable error to the user instead of a cryptic Ollama connection failure.
Output: validateConfig helper in helpers.go, call sites in root.go and inspect.go, tests in helpers_test.go.
</objective>
