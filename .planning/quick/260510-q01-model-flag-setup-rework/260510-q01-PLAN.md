# Quick Task 260510-q01: Model Flag & Setup Rework

## Goal

1. Add a `--model <name>` persistent flag to all commands so users can override the configured model per-invocation.
2. Rework `setup` model selection: list available Ollama models, let user pick one or pull a new one — remove the auto-hardware-detect-and-recommend flow from the active path (keep functions, just don't call them).

## Task 1 — Add `--model` persistent flag (`cmd/root.go`)

**Files:** `cmd/root.go`

**Action:**
- Add `var modelFlag string` package var
- Register `--model` persistent string flag on `rootCmd`
- In `ApplyFlagOverrides`, if `modelFlag != ""` set `cfg.Model = modelFlag`

**Verify:** `go build ./...` passes; `myhelper --model llama3.2:3b "hi"` uses that model.

**Done:** `--model` flag wired and applied before config validation.

---

## Task 2 — Rework wizard model selection (`internal/wizard/wizard.go`)

**Files:** `internal/wizard/wizard.go`

**Action:**
- Add `listModels(endpoint string) ([]string, error)` — calls `GET /api/tags`, returns sorted model names.
- In `Run()`, replace the hardware-detect + recommend + auto-pull block with:
  1. Call `listModels(endpointValue)` to fetch locally available models.
  2. If models exist: display numbered list (`[1] llama3.2:3b`, `[0] Pull a new model`).
  3. If user picks a number from the list: save that model as default, skip pull.
  4. If user picks 0 (or no models available): prompt for model name → pull via `pullModel`, then save.
  5. `detectMemoryMiB()` and `recommendModel()` stay in the file but are no longer called from `Run()`.

**Verify:** `go build ./...` passes; `go test ./internal/wizard/...` passes.

**Done:** Setup lists real models, no auto-hardware detection in the active flow.
