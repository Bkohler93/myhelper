<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → BuildArtifacts | File paths from Walk() feed into ExtractSymbolsFull; malicious .go files could contain extreme AST structures |
| BuildArtifacts → .myhelper/ | Writes JSON artifacts to a local directory owned by the process |
| chatFn response → project.json | LLM-generated summary stored verbatim in project.json |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | artifacts.go BuildArtifacts file write | accept | Writes only to `.myhelper/` under the project root (os.WriteFile with fixed relative paths); no user-supplied path is used in the write path |
| T-10-02 | Information Disclosure | symbols.json | accept | Contains only exported symbol names and signatures — no secrets. Stored locally, not transmitted |
| T-10-03 | Denial of Service | ExtractSymbolsFull on adversarial .go file | accept | go/parser is bounded by file size; the project is a local dev tool, not a service accepting untrusted input |
| T-10-04 | Tampering | project.json summary field (LLM output) | accept | Summary is generated from local summaries files; the LLM output is stored as a plain string; no eval or exec on LLM output |
| T-10-05 | Spoofing | schemaVersion field | accept | schemaVersion is a hardcoded constant "1.0" set at marshal time; not user-controlled |
</threat_model>
